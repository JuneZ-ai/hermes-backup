# 两步消化工作流 — P0 模板

> **灵感来源：** LLM Wiki 的两步思维链摄入（分析→生成）。
> **定位：** 将「手动消化素材」升级为「AI预分析 → 你审核 → AI生成知识条目」。

## 工作流总览

```
原始素材 (01-信息流/status:raw)
    │
    ▼
┌─ 第一步：LLM 预分析 ──────────────────┐
│ 输入：原始素材全文                      │
│ 输出：6块结构化分析                     │
└────────────────────────────────────────┘
    │
    ▼
┌─ 第二步：你审核（3-5 分钟）───────────┐
│ □ 实体提取准确？                       │
│ □ 跨模块关联有无遗漏？                 │
│ □ 矛盾点是否真实？                     │
│ □ 推荐结构是否合理？                   │
└────────────────────────────────────────┘
    │
    ▼
┌─ 第三步：LLM 生成知识条目 ────────────┐
│ 输入：素材 + 审核后的分析              │
│ 输出：知识条目 + index更新 + status     │
└────────────────────────────────────────┘
    │
    ▼
01-信息流/{素材}.md (status → processed)
```

## 第一步 Prompt

你以下面这份素材为输入，做「预分析」。

### 素材内容

```
{粘贴原始素材全文}
```

### 分析要求

输出以下 6 个区块（无内容则写「无」，不要硬编）：

**区块 1：关键实体** — 人物、组织、产品、事件。每个给出一句话定位 + 与六韬已有知识的关联。

**区块 2：核心概念** — 理论、方法、框架。每个给出定义 + 标注这是新概念还是对已有概念的补充/反驳。

**区块 3：跨模块关联** — 从以下模块选 1-3 个，说明理由：智脑/史鉴/易哲/决断之桥/太极双螺旋/wiki/实战案例。

**区块 4：矛盾与张力** — 与六韬现有知识库不一致的地方？冲突程度如何（修正/补充/对立）？

**区块 5：推荐结构** — 建议创建几个条目？每个的标题、编号、类型？建议的 [[wikilinks]]？

**区块 6：疑惑** — 不理解的点、需人工判断的问题、推荐的佐证搜索关键词。

## 第二步：审核清单

```
□ 实体提取：有没有重要实体被遗漏？
□ 概念定义：有没有过度解读或歪曲原意？
□ 跨模块判断：模块归属合理吗？
□ 矛盾点：你说同意的冲突的地方你同意吗？
□ 推荐结构：条目数量/编号/类型合适吗？
□ 疑惑项：AI说不确定的，你心里有没有答案？
```

## 第三步 Prompt

基于以下素材和我审核后的分析报告，生成知识条目。

### 原始素材

```
{粘贴原始素材全文}
```

### 审核后的分析

```
{粘贴你审核/修改后的分析报告}
```

### 生成要求

1. 创建 {建议路径}.md，含完整 frontmatter（title, tags, aliases, created, type, source, status:processed）
2. 正文结构：核心论点 → 关键概念 → 跨模块关联 → 六韬视角评价 → 延伸思考
3. 嵌入 [[wikilinks]] 指向已有知识条目
4. 更新模块 index.md
5. 更新原始素材 status: raw → processed

## 快速消化（< 2000 字短素材）

```
消化这份素材 → 创建知识条目 + 更新 index.md。
在创建前用 <analysis> 块输出你的预分析给我确认。
```
