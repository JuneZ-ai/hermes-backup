# 上下文预算规则

> 当需要 LLM 基于整个知识库回答问题时，显式控制上下文窗口分配。

## 核心公式

```
安全窗口 = 模型上限 × 0.8

分配:
  60% → 知识条目（相关页面全文）
  20% → 聊天历史（最近 N 轮）
   5% → 导航索引（总纲 + index.md）
  15% → 系统提示（SCHEMA + 格式规则）
```

知识条目部分再分：
- 50% → 关键词匹配的顶优先页面
- 30% → 一跳图谱邻居
- 20% → 二跳图谱邻居

## 各模型速查

| 模型 | 总窗口 | 安全(×0.8) | 知识(60%) | 历史(20%) | 索引(5%) | 系统(15%) |
|------|--------|-----------|-----------|-----------|----------|-----------|
| DeepSeek V3 | 64K | 51K | ~30K | ~10K | ~2.5K | ~7.5K |
| DeepSeek V4 Pro/Flash | 128K | 102K | ~61K | ~20K | ~5K | ~15K |
| Claude Opus 4 | 200K | 160K | ~96K | ~32K | ~8K | ~24K |
| GPT-4o | 128K | 102K | ~61K | ~20K | ~5K | ~15K |

## 场景预算建议

| 场景 | 预算 | 理由 |
|------|------|------|
| 快问快答（查概念） | 4K-8K | 1-2 页 wiki 概念 |
| 跨模块综合 | 16K-32K | 2 个模块各 3-5 页 |
| 全局扫描 | 64K-128K | 多个模块 index + 摘要 |
| 融合笔记 | 128K-200K | 跨模块深度阅读 |
| 知识缺口诊断 | 全量 | 尽量看全 |

## 标准 Prompt

```
## 上下文预算约束

当前模型上限 {N} tokens。按以下分配：
知识条目: {60%} tokens（最多 {X} 页）
聊天历史: {20%} tokens（最近 {M} 轮）
导航索引: {5%} tokens
系统提示: {15%} tokens

## 检索策略

1. 读取导航索引：00-六韬总纲.md + index.md + 相关模块 index
2. 按关键词从以下模块检索：智脑/史鉴/易哲/决断之桥/螺旋/wiki/实战
3. 加载匹配页面全文，按关联度排序，超预算则舍去低相关页面
4. 回答引用格式：[1] {页面名}，末尾附引用列表

## 规则

- 超预算时优先保留匹配度最高页面
- 某模块无相关内容则明确说明
- 禁止凭空创建不存在的 [[wikilinks]]
```
