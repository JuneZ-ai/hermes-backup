# Gap Analysis Prompt — P1+ 模板

> **灵感来源：** GBrain 的 `gbrain think` synthesis + gap analysis。
> **作用：** 在 LLM 回答末尾自动输出「已覆盖 + 知识空白 + 填补建议」三个区块。
> **原理：** 知道自己不知道什么，比知道自己知道什么更重要。

## Prompt 后缀

在正常提问末尾粘上这段：

```
---

回答时请同时在结尾附上以下三个区块：

## 📖 已覆盖
（你从知识库中找到了哪些相关页面？按模块分类列出）

## ❓ 知识空白
（以下哪些方面没有找到相关信息？）
- 区分：是「没人写过」还是「已有但没被检索到」
- 标注严重程度：严重/中等/轻微

## 💡 填补建议
（基于空白，建议从哪里补充？）
- 如需外网搜索：建议关键词
- 如不需要填补：说明理由
- 如需手动喂料：建议素材方向
```

## 示例

**提问：**
```
六韬知识库里，灰度决策有哪些不同模块的视角？
---
附带知识空白报告
```

**LLM 回答末尾：**

```
## 📖 已覆盖
| 模块 | 页面 |
|------|------|
| 六韬智脑 | ZN-02 灰度三要素 |
| 六韬易哲 | 易经/02 阴阳此消彼长 |
| 决断之桥 | QL-04 灰度×易理融合 |
| 六韬史鉴 | DS-01 知其白守其黑 |

## ❓ 知识空白
| 空白 | 程度 | 说明 |
|------|------|------|
| 灰度反面案例 | ⚠️ 中等 | 讲了方法论，没讲失败的案例 |
| 灰度×组织规模 | ⚠️ 中等 | 华为经验，没提小团队应用 |
| 灰度量化模型 | ⚠️ 轻微 | 缺少量化评估 |

## 💡 填补建议
| 优先级 | 方向 | 建议 |
|--------|------|------|
| P1 | 反面案例 | 搜索"灰度管理失败案例" |
| P2 | 小团队灰度 | 喂料相关主题素材 |
```

## 快速激活

在 system prompt 预置空白报告格式要求后，只需一句话：
```
{问题} + 附带知识空白报告
```

## 使用效果

随着空白报告逐渐变短，最终只剩「没有空白」——那时知识库的织网就织得差不多了。
