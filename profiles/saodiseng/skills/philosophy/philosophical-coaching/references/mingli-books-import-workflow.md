# 命理典籍入库工作流

## 触发条件

当用户提供命理八字类经典书籍文件（.md 或 .pdf），要求 "入库"、"融入第二大脑"、"加进知识库" 时，按以下流程处理。

## 文件来源

- 用户复制到 `C:\Users\<user>\Documents\` 的文件
- WSL 路径：`/mnt/c/Users/18502/Documents/`
- 格式：`.md`（已有文本）或 `.pdf`（需转换 PDF → MD）

## 入库步骤

### 1. 确认文件

用 search_files 在 Documents 目录搜索目标文件。

### 2. 检查文件体量

用 read_file 看前几行了解结构和大小。目标文件类型：.md 直接处理，.pdf 需先转 Markdown。

### 3. 添加 YAML Frontmatter

每本典籍入库前须添加标准 frontmatter，含 title, author, tags（八字 典籍 书名 六韬易哲）, created, source。

### 4. 确定目标路径

```
六韬易哲/八字命理/典籍/{书名}.md
```

所有八字命理典籍统一归入 典籍/ 子目录，不分拆（除非原书已有清晰分卷结构）。

### 5. 更新索引文件

入库后须更新以下两个索引文件：

**八字命理 MOC**（`六韬易哲/八字命理/八字命理.md`）：
- 在书单表格中添加状态列（✅ 已入库 / 待入库）
- 在典籍原文区块添加链接

**知识库总览**（`六韬易哲/YZ-00-知识库总览.md`）：
- 在推荐书单表中标记为 ✅ 已入库

### 6. PDF 转 Markdown

依赖：`pip install pymupdf`
脚本参考：`/tmp/pdf2md.py`
