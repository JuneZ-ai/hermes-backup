### ⚠️ 身强身弱判断（核心修正，2026-06-01）

**错误教训**: 第一版 bazi.py 直接从 BAGE 表取的通用喜忌（如伤官格→喜印/财），没判断身强身弱。用户指出用神应该是喜水喜木，不是喜印/财。根源是通用 BAGE 表没考虑身强/身弱。

**正确流程**: 赋值用神前必须先 run `judge_shenqiang()`，根据结果选择修正喜忌表。

#### 身强身弱加权评分算法 (`judge_shenqiang()`)

四个维度加权，正值=身强，负值=身弱：

| 维度 | 权重 | 细则 |
|------|:----:|------|
| **月令状态** | ±3~±0.5 | 当令+3，相(令生我)+2，休(我生令)-1，囚(令克我)-0.5，死(我克令)-2 |
| **地支根气** | +0.3~+1.5/个 | 同五行+1.5，生我五行+0.5，藏干本气外+0.3 |
| **天干印比/克泄** | ±1.0~±0.5/个 | 比劫+1，印星+1，官杀-1，食伤-0.5 |
| **年柱环境** | ±0.5 | 年支同五行+0.5，克我-0.5 |

**评分分段**:
- ≥ 2.0 → 身强
- +0.5 ~ +1.9 → 偏强
- −0.4 ~ +0.4 → 中和
- −1.9 ~ −0.5 → 偏弱
- ≤ −2.0 → 身弱

#### 身强身弱加权评分算法（代码格式）

备查：也可用以下 Python 伪代码表示 `judge_shenqiang()`：

```python
score = 0.0

# 1. 月令状态
if 月令五行 == 日主五行:  score += 3   # 当令
elif 令生我:              score += 2   # 相
elif 我克令:              score -= 2   # 死
elif 我生令:              score -= 1   # 休
elif 令克我:              score -= 0.5 # 囚

# 2. 地支根气
for each 地支:
    if 地支五行 == 日主五行: score += 1.5
    elif 地支五行 生 日主五行: score += 0.5
    for each 藏干:
        if 藏干五行 == 日主五行: score += 0.3; break

# 3. 天干印比 vs 克泄
for each 天干（不包括日干）:
    if 比劫:  score += 1.0
    elif 印星: score += 1.0
    elif 官杀: score -= 1.0
    elif 食伤: score -= 0.5

# 4. 年柱特殊
if 年支五行 == 日主五行: score += 0.5
elif 年支克日主:         score -= 0.5
```

#### 典型案例：乙木命盘评分

用户命盘（乙木，巳月，辛酉年，丙子时）：
- 月令巳火→木休囚: **-1**
- 日支卯为强根: **+1.5**，时支子水生木: **+0.8**
- 年干辛杀克身: **-1**，月干癸印生身: **+1**，时干丙伤泄身: **-0.5**
- 年支酉金克身: **-0.5**
- **总分约+0.3 → 中和偏弱**

结论：喜水（印）、木（比劫），忌金（官杀）、土（财）、火（食伤）

#### 特别注意：全盘缺土的案例

八字中年柱辛酉（金），月柱癸巳（水+火），日柱乙卯（木），时柱丙子（火+水）。
当全盘无土（财星缺位）+ 日主身弱时，忌神不仅包括官(金)杀，还包括财(土)和食伤(火)。用户说"都说我的喜用神是喜水喜木"——水和木辅助身弱日主（印和比劫），金火土都是忌神。这个判断跟通用模板不一样，是身弱判断修正的结果。

## 真太阳时修正 (2026-05-31 session)

### 案例: 用户生辰

```
农历: 1981年五月初五 01:48
出生地: 湖南吉首市 (109.7°E)
真太阳时修正: (120° - 109.7°) × 4分 = -41.2分钟
修正后: 01:07 → 丑时 (1:00-3:00)
```

如果用户在北京出生 (116.4°E)，修正只有 (120-116.4)×4 = -14.4分钟，影响不大。但吉首靠西，-41分钟刚好在子时→丑时的边界上 (00:00-01:00=子时, 01:00-03:00=丑时)。

### 边界检查规则

```
时辰边界: 子(23-1) 丑(1-3) 寅(3-5) 卯(5-7) 辰(7-9) 巳(9-11)
          午(11-13) 未(13-15) 申(15-17) 酉(17-19) 戌(19-21) 亥(21-23)

关键: 真太阳时后检查是否跨越了"整点"边界
- 如果原始时间在 01:00 ± 45分钟范围内 → 必须验证真太阳时后是否跨过01:00
- 如果原始时间在 23:00 ± 45分钟范围内 → 同上
- 其他时段一般不跨边界
```

### weipan.py 模板

每次用户给农历日期+出生地，写一个 weipan.py 做预处理:

```python
from lunar_python import Lunar, Solar

# Step 1: 农历→公历
lunar = Lunar.fromYmd(1981, 5, 5)  # 辛酉年五月初五
solar = lunar.getSolar()
print(f"公历: {solar.getYear()}-{solar.getMonth()}-{solar.getDay()}")

# Step 2: 真太阳时修正
longitude_correction = (120 - 109.7) * 4  # -41.2分钟
corrected_minutes = (1 * 60 + 48) + longitude_correction  # 01:48 → 约01:07

# Step 3: 调用排盘
from bazi import paipan
result = paipan("1981-06-06", "01:06", "male")
```

### 边界争议处理

当修正后的时间恰好靠近时辰边界（±15分钟内），在解读中同时说明两种可能性：

```
"先勘误：你的出生时间真太阳时修正后为01:07，落于丑时。
但如果出生地的经纬度稍有误差，或夏令时/真太阳时算法不同，
也有可能落在子时（丙子）与丑时（丁丑）之间。
我用丁丑时给你拆，但子时的情况差别是：……"
```

这样既诚实，又覆盖了不确定性。
