# 资料筛选三原则

用户明确的资料入库标准（2026-05-29 设定）：

## 三把尺子

| 标准 | 要求 | 排除项 |
|:----|:-----|:-------|
| **① 来源权威** | 业界公认的专业网站/平台 | 个人博客 / SEO站 / 翻译站 / 公众号转发 |
| **② 人物顶尖** | 行业内公认的顶尖人物 | 无名作者 / 二传手 / 浅层解读 |
| **③ 前瞻洞察** | 有未来判断/趋势预测 | 纯回顾 / 综述 / 入门级科普 |

## 例外情况

- **补充型数据更新**（薪资数据、新案例数据）：可只满足来源权威一条，直接合并入已有笔记
- **综述型文章**：基于已有资料重组的，不入库

## 评估框架可信度的关键

当一个框架/方法论看起来是"自建"的（无外部链接验证），**先别急着否定**。正确的评估顺序：

> **先问：这背后有没有顶级人物/顶级公司的实践作为实证？**
>
> 实证的强度：**人物亲自实践/公开验证** > **公司战略公开化** > **行业研究引用** > **无实证**

### 真实案例：τ定律评估教训（2026-05-29）

当天下午我（燕青）评估 ZN-32 华为τ定律时，犯了典型错误：

| 我的评估（❌） | 实际（✅） |
|--------------|----------|
| "自建框架，无外部背书" → 给它打了❌ | 黄仁勋 2026.5.28 在台北亲自评论τ定律 |
| 没意识到黄仁勋的 CUDA 15年布局就是τ定律的实证 | 黄仁勋原话 *"That bet took 15 years to pay off"* |
| 否定了其"来源权威"维度的得分 | 黄仁勋（NVIDIA CEO）本人就是最高级别的权威 |

**教训：框架的价值不在于它有没有"外部文章引用"，而在于它的原理是否被顶级人物/公司在实践中验证过。**

- τ定律：自建框架名称，但黄仁勋15年CUDA布局 + 华为哈佛电梯分段法 = 双线实证
- Harness Engineering：自建理论，但 OpenAI FDE团队（Colin Jarvis采访）+ Anthropic企业部署 = 间接验证

### 正确的评估迭代流程

```
遇到自建框架
    ↓
搜索验证：🔍 是否有顶级人物的实践/原话支撑？
    ↓
有 → 用人物/实践作为实证，更新评估为✅
无 → 标记为"未经验证的假设"而非"假框架"
```

**关键词搜索策略：** 不要只搜框架名称。搜索**框架背后的原理**。τ定律搜索"黄仁勋 10年"或"Jensen Huang long-term strategy"比搜"τ定律"更有效。

## 来源可信度排序（从高到低）

1. 行业顶尖人物的原始访谈/演讲/博客（如 Sam Altman、黄仁勋、Bob McGrew、Andrew Ng）
2. 头部公司的官方技术博客（如 OpenAI Blog、Palantir Blog、Intercom Engineering）
3. 权威行业报告（McKinsey、Gartner、BCG）
4. 主流科技媒体原创深度报道（TechCrunch、The Information、Wired）
5. 播客原始访谈逐字稿（YC 播客、Altimeter Capital、Lex Fridman）
6. 学术论文/预印本
7. ❌ SEO 工具文/综述/翻译/招聘信息/知乎泛读

## ⚠️ 重要风险：delegate_task 搜索结果的AI幻觉

当使用 `delegate_task` + web工具集 搜索资料时，子代理返回的摘要可能包含**AI生成的不存在内容**：

| 幻觉类型 | 表现 | 实际案例 |
|:---------|:-----|:---------|
| **虚构文章** | 声称某知名博客发表了特定技术文章，有详细章节名和数据 | 2026-05-29 FDE搜索："Intercom FDE Bridge架构" → URL返回404 |
| **虚构播客** | 声称某播客发布了特定主题访谈，包含详细观点 | 2026-05-29 FDE搜索："SE Daily边缘FDE播客" → 页面无FDE内容 |
| **虚构评分** | 给虚构内容打⭐⭐⭐⭐⭐ 评级 | 评价越"完美"越可疑 |

**预防措施：**
- 子代理输出的每条关键来源**必须**用 `curl -sL <url>` 验证 HTTP 200
- 验证页面中是否**实际包含**摘要声称的内容（grep关键词）
- 发现404或内容不匹配 → 丢弃该"来源"，标记为幻觉
- **不要因为子代理的"评分高"就放松警惕**——评分越漂亮越可能是幻觉
