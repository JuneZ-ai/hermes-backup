## 评审发现（关键修正）

### 1. ❌ 不要物理搬文件
**最大盲区。** 百年孤独在物理上同时属于：
- 易哲/文史哲的「文史通识链」（中国简史→文学简史→拉美文学）
- 道史的「孤独哲学链」（百年孤独→一句顶一万句→说话的哲学）

**方案改：** 接受「一物双归属」，用别名+标签+MOC索引解决，不用文件路径强制规范。总纲改映射即可，不搬文件。

### 2. ⚠️ 物我合一分身
`六韬智脑/19-埃森哲AI物我合一框架`（实用方法论版）和 `六韬易哲/物我合一`（哲学解读版）同时存在。保留双版本，总纲标注关系。

### 3. ✅ 使用场景速查保留
三大开关入口不能覆盖「随便逛逛学点东西」场景。两种入口应共存。

### 4. ✅ 命名优化
保持「六韬智脑」「六韬易哲」等名称为主标题，「X之道」为副标题/别名，不强制改名。

### 5. ✅ 决/断 → 理/觉
原方案核心弱点：决和断在中文里连用为"决断"，边界模糊。改为理（理性拆解）和觉（直觉感知）后语义距离极大，不会混淆。

## 实际实施：六韬之树（Hermes-Junis 最终版）

Hermes-Junis 没有用「道→法→术」的建筑隐喻，而是提出了**树形隐喻**：

```
                    🚀 行 · 实战案例（树冠：开花结果）
                        ↑
                    ⚖️ 融 · 决断之桥（枝干交汇）
                      ↗         ↖
              🧠 理 · 智脑     💭 觉 · 易哲（枝干：向光生长）
                      ↖         ↗
                    📖 鉴 · 道史（主干）
                          ↑
              🌳 太极双螺旋（根系：向下汲取，向上供养，破土成枝）
```

**关键创新：** 破土成枝概念——自心洞察、纳瓦尔幸福哲学、狂热心理学不是从主干（道史）或枝干（智脑/易哲）分出来的，而是从根系（双螺旋）直接发芽长出的新枝干。这解释了为什么它们有自己的模块编号（LX-07/08/09）。

### 树vs建筑对比

| 建筑（原方案） | 树（最终版） |
|:---|:---|
| 楼层固定，建好就不动了 | 树一直在长——根系深扎、枝干伸展、树冠更新 |
| 上层依赖下层，单向支撑 | 根系供养分向上，果实落回土壤——双向循环 |
| 地基埋在底下不动 | 根系主动生长——向下扎更深，向上还会破土成枝 |
| 楼层之间是分界 | 枝干之间可以交叉——决断之桥就是嫁接点 |

### 新编号系统

Hermes-Junis 建立了统一的模块前缀编号：
- LX = 螺旋（太极双螺旋） 
- DS = 道史
- ZN = 智脑  
- YZ = 易哲
- QL = 桥梁（决断之桥）

## 关键教训

- 知识库的真正灵魂是跨模块链接网络，不是树形层级
- 一物可以有双归属，这不是设计缺陷而是信息过载的自然后果
- 任何文件物理搬迁操作前，必须检查跨模块链接引用
- 重构分阶段实施，先零风险后高风险
- **建筑隐喻不如树隐喻**——知识库是活的，应该用活的隐喻
- 重构方案必须提交 Hermes-Junis 评审，不能直接动手
