# 橙皮书/长文排版风格（花叔风格）

> 适用于需要输出完整手册/成长指南/知识体系文档的场景。源自花叔《Hermes Agent 从入门到精通》橙皮书 v260407。

## 总体结构

```
Cover Page ─→ TOC ─→ Parts (§01-§N) ─→ 后记
```

## Cover Page 格式

```
橙皮书
[标题]
从入门到精通

[副标题/一句话定位]
[关键词说明]

关键词：[逗号分隔]
适合读者：[逗号分隔]

版本：[vX.X]
[作者/来源信息]

[版本脚注]
```

## 目录 (TOC) 格式

```
目录
CONTENTS

Part 1: 中文名
§01
中文标题
English Title: English Subtitle
§02
中文标题
English Title: English Subtitle

Part 2: 中文名
...
```

- Part 编号用阿拉伯数字
- §XX 编号独立，不分 Part 重新编号
- 英文标题冒号分隔，跟在中文标题下一行

## 章节格式

### 章节标题

```
§XX  中文标题
English Title: English Subtitle
```

正文开始。每个章节头部的英文标题用冒号分隔，跟在中文标题下一行。

### 正文风格

- **口语化开场**：用问题/观察/故事开头（"又来了一个？""我理解你的疲惫。"），不要学术化的"本文将..."
- **框架/流程图**：优先用 ASCII art 或 mermaid，不要用截图
- **数据表**：用 markdown 表格，第一列加粗
- **关键概念**：粗体标出
- **金句引用**：`> "原话引用"` blockquote + 署名
- **对比表**：行对比优于列对比，维度在左，对比项在右

### "60秒看懂"页面设计（橙皮书标志性元素）

每本橙皮书的 §02 或其他概述章节使用**三区块布局**：

```
第一区块：架构一张图（ASCII box）
    ┌──────┬──────┬──────┐
    │ 模块A │ 模块B │ 模块C │
    └──────┴──────┴──────┘

第二区块：关键数据表
| 指标 | 数据 | 时间/备注 |
|:----|:----|:----------|

第三区块：对比表（两个概念的核心差异）
| 维度 | 选项A | 选项B |
|:----|:------|:------|
```

这个布局让读者60秒内建立全局认知。三区块固定顺序：架构图→数据表→对比表。

### 案例格式

```
**背景**：一句话
**问题**：一句话
**FDE解法**：具体怎么做
**成果**：量化数据
```

## 语言风格

- **第一人称"我"** — "我理解你的疲惫"、"我花了一周时间"
- **口语化但不失严谨** — 可以有"对"、"换句话说"、"到底"等口语连接词
- **英文术语自然混入** — 不需要翻译已通用的术语（FDE / SWE / SaaS / RAG / API）
- **数字用阿拉伯数字** — 不用汉字数字（除非是成语/固定搭配）
- **强调用粗体** — 不用下划线或斜体
- **列表用 ` | ` 管道符** — 不用 `-` 或 `*` 作为表格边框
- **后记收尾**：花叔金句 + 同格式的本领域改编版
