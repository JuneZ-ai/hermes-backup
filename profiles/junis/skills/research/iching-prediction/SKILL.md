# 双轨预测法（易哲 + 实证）

> 卦为体，数为用。体用兼赅，始得真机。

## 工作流程

### 第一步：收集素材（数据轨道）

在起卦之前，先获取足够的事实信息：

```
赛事/事件类型 → 基本信息（时间/地点/参与方）
  → 历史数据（过往交手记录、同类事件历史规律）
  → 当前态势（近期表现、伤病/变动/环境因素）
  → 结构化摘要（供后续分析引用）
```

网络受限时，优先使用可通渠道：
- Bing / Sogou / 百度（中文）
- Oddsportal（体育赔率，需 curl + 合理的 UA）
- 微信公众号文章（微信搜狗）
- 体育数据站（500.com / 搜狐体育等）

### 第二步：起卦推演（易哲轨道）

用当前时间（或指定事件时间）的农历信息起卦：

1. **获取农历数据**：`lunar-python` 库（已预装）
   ```python
   from lunar_python import Solar
   s = Solar.fromYmd(year, month, day)
   l = s.getLunar()
   ```

2. **梅花易数起卦法**：
   - 上卦 = (年+月+日) mod 8
   - 下卦 = (月+日+时) mod 8
   - 动爻 = (年+月+日+时) mod 6
   - 八卦数值：1乾 2兑 3离 4震 5巽 6坎 7艮 0坤

3. **分析层次**：
   - **体用生克**：体卦（主方）vs 用卦（客方）的五行关系 → 大势判断
   - **动爻爻辞**：动爻的爻辞 → 关键变数信号
   - **变卦**：动爻变后的卦 → 发展走向
   - **互卦**：本卦中间四爻 → 中间过程

### 第三步：市场实证（验证轨道）

**预测必须有交叉验证。单靠卦象结果不可提交。**

数据源优先级：
1. **博彩赔率**（体育赛事）：Oddsportal / 500.com / 澳客网
   - 1X2（胜平负）赔率 → 反推市场隐含概率
   - 亚洲让球盘 → 市场对"实力差"的判断
   - 赔率变化趋势 → 资金流向信号
2. **预测市场**（非体育）：Polymarket / Kalshi 等（如有skill可用）
3. **专家共识**：搜索行业分析、赛前评论、媒体前瞻

将市场隐含概率与卦象判断对比：
- 一致 → 信心增强
- 不一致 → 不可忽略警告，修正预测

### 第四步：综合输出

输出格式（三栏对比）：

```
卦象推演（天时）   |   市场数据（人和）   |   综合结论
------------------|--------------------|-----------
体克用→主方有利   |   赔率2.50 vs 3.20  |   主队不败但非稳赢
鹤鸣子和→平局可能 |   平赔3.00偏低     |   平局是最高价值选项
```

### 第五步：可选的案例入库

如果需要积累案例素材，将完整预测记录为笔记：
- Obsidian: `01-信息流/易哲实战案例-<主题>.md`
- 含：起卦方法、卦象分析、数据来源、最终结论、事后验证（赛后补充）

## 坑 / 关键注意

### 绝对不要
- ❌ 单靠卦象结论提交 → 缺少"人和"维度的预测是半成品
- ❌ 无视市场赔率的强烈信号（如主胜赔率>2.50却说稳赢）
- ❌ 用"天机不可泄露"之类的模糊话术敷衍
- ❌ 把博彩数据当绝对真理（市场也会错，但它是已知最好的校准器）

### 一定要
- ✅ 明确标注数据来源和时间戳
- ✅ 区分"卦象指向"和"市场信号"
- ✅ 给出具体比分/数值判断（如"2-1最可能"而非"胜负难料"）
- ✅ 标注置信度：卦象一致 + 数据一致 → 高；有分歧 → 低

### 环境注意
- Oddsportal 需要合理的 UA 和 `--compressed` 参数
- 国内博彩站（500.com等）可能被 GFW/DNS 干扰——优先试 Sogou / 百度搜索结果中的片段
- 动态 JS 渲染的赔率页不起卦的直接起卦——先用搜到的文字片段
