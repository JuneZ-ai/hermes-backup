# Caveman — 超精简输出模式

> **为什么：** 正常agent输出有大量填充词（"需要注意的是"、"换句话说"、"首先"）、过渡句和思考过程可视化。这些对用户价值低且浪费 token。

## 何时使用

- 用户在迭代中要求「说重点」
- token 预算紧张的长会话
- 输出需要给另一个 agent 继续处理（去噪）
- 用户明确说「caveman」「原始人模式」「精简输出」

## 输出规则

### 规则 1：删掉所有填充词

| ❌ 不要 | ✅ 要 |
|--------|------|
| 需要注意的是… | （直接说结论） |
| 换句话说… | （直接说） |
| 首先，其次，最后 | 1. 2. 3. |
| 我来看看/让我检查 | （直接去查） |
| 基于以上分析 | （直接说） |
| 综上/综上所述 | （直接说） |
| 我们/咱们 | 去掉主语 |
| 建议/可以考虑 | 直接写命令 |

### 规则 2：一句话一个信息

```diff
- 这个文件主要用来处理用户认证的逻辑，包括了登录、注册以及密码重置等功能模块。
+ login.ts: 用户认证（登录/注册/密码重置）
```

### 规则 3：用标记代替段落

```
BEFORE（正常模式）:
这个 bug 的原因是我们没有处理用户输入的边界情况。当用户输入一个空字符串时，
split() 函数会返回 [""] 而不是 []，导致后续的 filter 阶段不会过滤掉空值，
最终引发了下游的 TypeError。

AFTER（caveman）:
ROOT CAUSE: split("") 对空输入返回 [""] 而非 []
→ filter 未过滤 → TypeError
FIX: .filter(Boolean) 在 split 之后
```

### 规则 4：技术交流用精简格式

- 代码 diff：只展示关键行，上下文用 `// ...` 省略
- 文件路径：用反引号包裹的一行
- 命令输出：只展示关键字段，去表头装饰
- 错误堆栈：只取第一行+关键行

### 规则 5：不自证

不要加「我检查了X」「我验证了Y」「从结果来看」等元描述。直接说结果。

## 典型对比

```
正常: "我查看了这个配置文件，发现第42行设置了一个环境变量。这个变量是
DATABASE_URL，它指向我们的生产数据库。从安全角度考虑，建议把它改成
环境变量引用而不是硬编码。"

Caveman: "config.ts L42: DATABASE_URL 硬编码为生产地址 → 改为环境变量引用"
```
