## 一、经验全景

```
PDF类型检测 → 文本型 → PyMuPDF → 秒级
           → 扫描型 → Tesseract OCR(150dpi) → 3-4秒/页
           → 混合型 → 文字层+OCR图片取交集

多格式输入 → DOCX → python-docx → 自动章节检测
           → EPUB → ebooklib（未实战）
           → TXT/HTML → 直接读取/BS4解析

DPI选择   → 150dpi = 甜蜜点（速度质量平衡）
         → 200dpi = 古籍推荐（6-8秒/页）
         → 300dpi = 仅印刷质量差时（10-15秒/页）
```

## 二、OCR质控经验

### 预处理效果实测
- **仅灰度化** = 基线（推荐大部分场景）
- 灰度+对比度增强(2x) = +0.3s/页, +2%质量
- 灰度+锐化 = +0.4s/页, +1%质量
- 灰度+二值化 = -3%质量（可能丢细节）
- 全预处理管线 = **不推荐**（性价比低）

### 后处理优先级
1. 中文间空格合并（Tesseract默认字间空格）
2. 多余空白行合并
3. 中英文标点统一
4. 常见OCR误识替换（如「埃棍哲」→「埃森哲」）
5. 领域专用字典（专有名词纠错）

### 已知OCR缺陷
- 加粗/大号字体：识别率下降20-30%
- 竖排文字：完全无法识别（需旋转图片）
- 浅色/反白文字：需图像反转预处理
- 表格边框：可能被识别为文字

## 三、文档后处理

### 现代文档深度阅读笔记工作流
```
OCR全文 → 章节自动检测 → 核心摘要 → 逐章要点提炼 →
跨章节洞察 → 知识体系关联 → 行动建议 → 金句摘录 →
Obsidian格式化（双链+标签+元数据）
```

### 章节自动检测策略
- 标题页：大号字体+独占整页+关键词（封面文章/特写）
- 章节标题：独立行+字号突变+冒号/破折号分隔
- 作者署名：标题后独立行+短文本（2-4字）
- 章节边界：新标题出现=上一章结束

## 四、一句话原则

1. **先检测类型，再选工具** — 文本型秒级，扫描型分钟级，错误选择浪费数倍时间
2. **150dpi是甜蜜点** — 再高不划算，再低质量崩
3. **预处理克制** — 灰度化足够，过度预处理是时间陷阱
4. **flush是你的朋友** — 后台任务不加flush等于盲飞
5. **古籍和现代文档是两条路** — 不可混用同一套处理逻辑
6. **踩坑记录比功能描述更有价值** — 后人不需要知道"能做什么"，需要知道"什么会炸"
