# Handoff — 会话交接

> **为什么需要交接文档：** 长会话中上下文越积越重。交接文档让下一个 agent 或人类在 30 秒内理解现状，不用重头问一遍。

## 何时使用

- 当前会话超过 30 轮，上下文开始紧张
- 要交给另一个 Bot（燕青→黄老邪，或反过来）
- 人类开发者要接手继续工作
- 跨 session 的长期任务，需要 checkpoint

## 交接文档模板

输出到 `~/.hermes/profiles/junis/output/handoff-{时间戳}.md`：

```markdown
# 交接文档

## 基本信息
- 任务：[一句话概括]
- 发起人：[Bot名/人名]
- 交接时间：{YYYY-MM-DD HH:mm}
- 当前状态：🔴 阻塞 / 🟡 进行中 / 🟢 待执行

## 已完成
- [x] [已完成事项 1]
- [x] [已完成事项 2]

## 待办
- [ ] [待办 1 — 优先级/依赖]
- [ ] [待办 2 — 优先级/依赖]

## 关键上下文
- 相关文件：[路径列表]
- 决策记录：[重要决策点]
- 阻塞项：[当前卡在什么地方，需要什么才能继续]

## 风险提醒
⚠️ [接手者需要注意什么]
```

## 精简版（30秒版）

当对方是 agent 时，可以用更紧凑的版本：

```
### Handoff: {任务名}
FROM: {当前agent}
TO: {目标agent}

DONE:
- {关键完成项}

TODO (按优先级):
1. [ ] {待办 1}
2. [ ] {待办 2}

BLOCKERS:
- {卡点}

FILES:
- {核心文件路径}

CONTEXT:
{3-5行关键背景}
```

## 附注

- 交接文档也可以写日志到搭建日志 Bitable 作为记录
- 如果任务跨 bot，建议在交接到目标 bot 时同时触发 `/handoff` skill 载入上一个交接文档
