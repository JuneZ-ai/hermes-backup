# 知识→智慧转化管道

> 口诀：**拆→连→筛→测→沉，三振补末端。**
> 一道工序对应一个思辨工具，五向哲学贯穿始终。

---

## 适用场景

- 用户投喂大量资料（采访实录、书籍、文章、论文等），需要系统吸收
- 需要判断一个知识/框架是否值得内化为默认行为模式
- 大规模内容消化（10万字+级别的采访/书籍/文档）

## 核心理念

**知识 ≠ 智慧。**

| 层次 | 定义 | 检验标准 |
|------|------|---------|
| **信息** | 原始数据/事实 | 接收即可 |
| **知识** | 结构化、关联过的信息 | 可检索、可引用 |
| **智慧** | 内化到行为层面的知识 | 遇到新问题时自动调用，不需检索 |

从知识到智慧，需要经过五步：**拆→连→筛→测→沉**。

## 五步转化管道

### Step 1: 拆解框架 → 用「第一性原理」+「体用如一」

拿到输入材料，先做三问拆解：

| 问题 | 判断 |
|------|------|
| 事实型还是框架型？ | 事实→存知识层。框架→有机会升DNA |
| 核心论点是什么？ | 一句话概括——概括不了就没真正理解 |
| 支撑逻辑是什么？ | 拆出论据链：因为A所以B，路径清晰吗？ |

**思辨工具**：
- **第一性原理**：剥离表象，直击本质。"它的源头公理是什么？"
- **体用如一**（道家）：什么是体（根本规律）？什么是用（具体表现）？

**产出**：材料标记 `fact` 或 `framework`。框架型材料决定是否进入Step2。

### Step 2: 连接已有知识库 → 用「系统思维」+「阴阳转化」

新知识和已有框架碰撞，判断三类关系：

| 关系 | 标记 | 处理 |
|------|------|------|
| **补充** | compatible | 已有一致，加深/细化 |
| **冲突** | conflict | **必须出立场决定**——要么新知识不对，要么旧框架需要更新 |
| **无关** | new_domain | 独立新领域，按标准流程入库 |

**思辨工具**：
- **系统思维**：不孤立看新知识——它和哪个框架相连？这个连接会改变什么全局？
- **阴阳转化**（道家）：冲突是"阳"，暴露的盲点是"阴"——在冲突中找升级契机

### Step 3: 对比筛选 → 用「批判性思维」+「墨家三表法」

五维评分决定晋升潜力：

| 维度 | 权重 | 高分表现 |
|------|------|---------|
| **相关度** | 30% | 直接解决当前1个以上真实痛点 |
| **通用度** | 25% | 可跨3个以上领域复用 |
| **一致度** | 20% | 与SOUL核心信念完全对齐 |
| **可操作度** | 15% | 有清晰的"下一步行动" |
| **时效度** | 10% | 长期有效，不过时 |

**阈值**：≥2.5 → 进入Step4 | 1.5~2.4 → 待观察 | <1.5 → 归档

**思辨工具**：
- **批判性思维**：甄别前提假设，找反例
- **墨家三表法**：三维交叉验证——历史经验（上本）、现实数据（下原）、实践效果（发以为用）

### Step 4: 行为测试 → 用「成长思维」+「诸行无常」

关键环节——不是马上改DNA，是知情观察：

1. 留心调用机会——遇到相关场景时主动问"这里能用这个框架吗？"
2. 记录第一次调用——效果如何？
3. **连续3次在合适场景自动调用** → 证明内化成功 → 晋升DNA
4. 一直没机会调用 → 场景不匹配，退回知识层

**思辨工具**：
- **成长思维**：测试失败不是"框架不行"，是"场景不行"——错误=优化数据
- **诸行无常**（佛学）：没有永恒正确的框架，即使晋升DNA也要保持"可能过时"的开放

### Step 5: 沉淀归档 → 用「逆向思维」+「为道日损」

无论是否晋升DNA，都要归档：

- ✅ **晋升DNA** → 写入对应DNA文件，更新总览
- ❌ **退回知识层** → 按入仓五维评分归档到ZN-永久笔记
- ⏳ **待观察** → 标记 pending_promotion，设定复查时间

**思辨工具**：
- **逆向思维**：反过来问——"如果不归档，会错过什么？""如果保留但从未引用，是不是认知负债？"
- **为道日损**（老子）：砍掉冗余，只留精华。归档也是"减"的机会。

## 和「三振晋升」的关系

| 管道 | 方向 | 触发 | 机制 |
|------|------|------|------|
| **五步转化** | 主动 | 用户投喂资料 | 拆→连→筛→测→沉 |
| **三振晋升** | 被动 | 用户指出错误 | 第3次同样错误后写入DNA |

两者构成完整的智慧生长系统：**从输入学 + 从错误学**。

### 五思一法

| 工具 | 本质 | 最适用步骤 |
|------|------|-----------|
| **系统思维** | 洞察动态关联与整体规律 | Step2 连接 |
| **批判性思维** | 空杯心态，甄别真伪 | Step3 筛选 |
| **第一性原理** | 剥离表象，直击本质 | Step1 拆解 |
| **逆向思维** | 反过来想，规避盲点 | Step5 归档 |
| **成长思维** | 错误是优化知识库的养分 | Step4 测试 |
| **合思法** | 双螺旋+多径原则+韬定律三问 | 复杂决策、跨模块分析 |

### 东方六器（DNA-02）

| 工具 | 本质 | 五步映射 |
|------|------|---------|
| **体用如一** | 透过现象求本质 | Step1 |
| **阴阳转化** | 在对立中找升级契机 | Step2 |
| **墨家三表法** | 三维交叉验证 | Step3 |
| **诸行无常** | 消解对任何框架的执念 | Step4 |
| **中道思辨** | 超越对立找更高维度 | 复杂决策时 |
| **知止而后有定** | 以终为始校准方向 | 启动前 |

## 大规模内容消化策略

当用户投喂 >10万字的内容（如8册采访实录+2本系统著作）：

### 分层策略

| 批次 | 做什么 | 工具 |
|------|--------|------|
| **第1-2册** | 完整扫描，找出核心框架 | read完整 |
| **后续册数** | Delta扫描——只找新东西 | 基于已有框架比对 |
| **系统著作** | 完整扫描，独立消化 | 如《熵减》作为元框架 |

### Delta扫描模式

已有基础框架后，后续内容不再全文重读，而是：

1. 扫描目录/TOC了解结构
2. 关键词搜索：用 search_files 搜索已有框架的关键词
3. 只读未出现过的主题/章节
4. 产出：新框架清单 + 补充说明 + 矛盾标记

### 大规模消化的参考数据

2026-06-08 实战（华为主题）：
- 8册采访实录 ≈ 1.6MB，约14,000行
- 熵减全书：449KB，5,709行
- 方法论以奋斗者为本：394KB，5,911行
- 灰度管理法：245KB，6,718行
- 总计：约2.7MB，约32,000行原始材料
- 产出：ZN-46（347行）+ ZN-47（456行），共约800行提炼框架

## Pitfalls

### ⚠️ 核实事实数量再开干

**用户给任务描述时，涉及数量/时间/金额/文件名等可验证信息，必须先核实再行动。**

错误示范（2026-06-08教训）：用户说"喂了4册"→未核实文件数→实际只收到2册→浪费token纠正。

**正确流程**：
1. 听到数量/时间/金额/文件名等
2. 先核实事实（列文件数、查系统状态）
3. 有偏差如实反馈，不为用户做解释
4. 一致再开跑

### ⚠️ 不要同时用多个subagent改同一文件

多册扫描时如果有并行task共享同一个文件，串行处理而非并行。

### 大规模内容消化策略与Delta扫描模式

当用户投喂 >10万字的内容（如8册采访实录+2本系统著作），采用分层策略避免重复劳动：

| 批次 | 做什么 | 工具 |
|------|--------|------|
| **第1-2册** | 完整扫描，找出核心框架 | read完整 |
| **后续册数** | Delta扫描——只找新东西 | 基于已有框架比对 |
| **系统著作** | 完整扫描，独立消化 | 如《熵减》作为元框架 |

**Delta扫描模式**：已有基础框架后，后续内容不再全文重读，而是：
1. 扫描目录/TOC了解结构
2. 关键词搜索：用 search_files 搜索已有框架的关键词
3. 只读未出现过的主题/章节
4. 产出：新框架清单 + 补充说明 + 矛盾标记

**框架间矛盾处理模式**：当两个框架对同一类问题给出不同指导时：
1. 记录矛盾，不回避
2. 找管辖范围——各自适用的阶段/条件
3. 找衔接点——什么时候切换
4. 不做强行统一

### 预搜索：Awesome列表先查

当进入不熟悉的领域时，先查 sindresorhus/awesome 精选资源索引（47万星），适用于领域探索型搜索，不适用于目标定位型搜索。

### ⚠️ 不要在大规模吸收中丢失用户偏好

用户偏好优先级：历史/经济>命理/中医。入库同分优先选历史经济。不确定的时间线宁可留空不猜测。

## 质量检查

- 收进来的知识 >70% 最终归档为"知识层"不是问题
- 连续3次以上被复述但不被自动调用 → 转化管道堵塞
- Step1-3永远快速通过从不筛选 → 缺乏批判性，不是"博采众长"而是"来者不拒"
- 反思循环中有工具从未被使用 → 工具箱里有闲置武器
