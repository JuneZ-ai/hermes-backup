# 桥笔记构建指南（Cross-module Bridge Note Template）

> 用于将跨模块对话沉淀为永久知识库桥接笔记。**核心不是对比，是发现同一张底牌的不同花色。**

## 触发条件

桥笔记不是每次跨模块回答都写。判定标准：

- ✅ 结构化对称：两个知识体系有可映射的结构（如克鲁伊夫角色互换→泰卦天地交泰）
- ✅ 关系型分析：可追溯的传承或对比（如毛邓同源异流）
- ✅ 用户主动要求"沉淀"——这是最高优先级信号
- ❌ 不沉淀：纯粹事实问答、一次性修辞、无结构性的类比

## 章节结构（已验证4次）

### 1. 核心洞察（一句话概括）
放在标题下方，一句话点明整座桥的底层逻辑。

### 2. 第一面镜鉴：X 框架展开（3-4节）
详细展开第一个知识体系的框架，用原文+案例+表格。每面镜鉴需要让读者独立理解这个体系。

### 3. 第二面镜鉴：Y 框架展开（3-4节）
同上述结构，但突出与第一面镜鉴的潜对应。

### 4. 第三/四面镜鉴：继续展开（3-4节）
逐一展开所有参与桥接的知识体系。每个镜鉴独立成章，但文字中保持与前一章的回钩。

### 5. 结构对称表（核心交付）
表格形式呈现，每个知识体系一列。不要泛泛对比，要逐行对应：
- 同一命题下各体系的不同回应
- 底层逻辑行总结各回应的共同本质

行数6-8行最佳，太短没深度，太长读不完。

### 6. 差异化定位表
与结构对称表互补——对称表讲对应，定位表讲差异。行维度：核心关切、解决问题、言说风格、适用场景、局限。

### 7. 对普通人的启示
3-4条可操作原则 + 一句话速记卡（每个体系一句）。每条原则要有具体行动指令，不止是道理。

### 8. 金句集
每个知识体系独立小节，每节4-6条，带出处。不做全量金句搬运，只选与桥主题相关的。

### 9. 知识库定位（ASCII图）
用 ASCII 树展示这座桥如何连接各模块：
```
模块A ← → 桥笔记 ← → 模块B
                  ↕
              模块C
```

### 10. 认知增量
放在最后，回答：这座桥给了什么新东西？读者看完后认知发生了什么变化？

## 写作规范

- title 格式：`NN-主题·子标题.md`
- tags 包含所有参与桥接的知识体系
- source 字段列出所有源素材（如 `《一句顶一万句》× 段永平哲学 × 纳瓦尔幸福哲学 × 《百年孤独》`）
- 保持每个镜鉴独立可读（读者不需要先看过源笔记就能理解）
- 语言比正文笔记更口语化——桥笔记是"说人话"的出口

## 已知问题

| 问题 | 表现 | 对策 |
|:----|:-----|:-----|
| 生硬类比 | "X和Y都很智慧"型总结 | 必须有结构性对称才入库，否则不写 |
| 差异化冗余 | 对称表和定位表有重复内容 | 对称表聚焦"对应"，定位表聚焦"差异"，不要写岔 |
| 插入新镜鉴后序号错乱 | 三重→四重后章节号对不上 | 每次增补镜鉴后重新检查所有章节编号 |
| 四重镜鉴后表格太宽 | 5列+的 markdown 表格阅读体验差 | 超过4个镜鉴时拆为两张子表或用分段呈现 |
