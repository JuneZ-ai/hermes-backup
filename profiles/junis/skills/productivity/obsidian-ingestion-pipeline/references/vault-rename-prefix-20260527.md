## 冲突解决规则

- 按文件修改时间判定"先来后到"
- 先创建的保持原编号，后创建的取下一个空闲序号
- 四对冲突按此规则全部解决（23/26/27 在智脑，07 在双螺旋）

## 执行脚本模式

### 第一阶段：替换 wikilinks + 重命名
1. 构建 `(模块, 旧文件名, 新文件名)` 映射表
2. 用 Python regex 扫描全库 `.md` 文件，同时匹配三种 wikilink 格式：
   - `[[目录/旧文件名\|显示]]` — 完整路径带别名
   - `[[目录/旧文件名]]` — 完整路径无别名
   - `[[旧文件名\|显示]]` — 同目录简写
   - `[[旧文件名]]` — 同目录简写无别名
3. 先替换所有文件中的 wikilink（读→改→写），再重命名文件
4. 特别注意：文件名中可能含 `.` `·` 等 Unicode 字符，需要 `re.escape()`

### 第二阶段：验证
1. 旧文件名全部删除（`find -name "旧名.md"` 返回空）
2. 新文件名全部存在
3. 总纲等关键文件不再有旧格式 wikilink
4. 跨模块链接双向验证（如 LX-12→ZN-23, ZN-23→LX-12）

## 注意事项
- 总纲中经常有 `\\\\`（四重反斜杠）和 `\\`（双反斜杠）转义符号，替换前必须预处理
- Obsidian wikilink 中的竖线 `|` 不需要反斜杠转义——之前总纲中的 `\\\\|` 是残留错误
- 易哲目录下的子库（易经体系/八字命理/中医基础等）里面的文件不需要重命名——只改根级
- 六韬易哲 MOC地图/ 下的文件也不需要前缀
