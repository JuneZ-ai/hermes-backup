# 源感知排名（Source-aware Ranking）

> 吸收自 GBrain（Garry Tan）的 `src/core/search/sql-ranking.ts`。原文在 `/mnt/d/龙虾技能/skill/gbrain-master/gbrain-master/`

## 核心理念

不是所有笔记在搜索时价值相同。不同模块的笔记在跨库引用时的权重应该不同。

## 权重映射（六韬知识树）

| 模块 | 权重 | 理由 |
|:----|:----:|:-----|
| **六韬智脑 (ZN-)** | **1.5×** | 方法论精华，引用价值最高 |
| **决断之桥 (QL-)** | **1.4×** | 跨模块合成物，桥接价值高 |
| **六韬易哲 (YZ-)** | **1.3×** | 哲学洞察，可提供认知框架 |
| **太极双螺旋 (LX-)** | **1.3×** | 底层思想范式，根基价值 |
| **六韬史鉴 (DS-)** | **1.2×** | 历史参照系，提供参照但不直接指导行动 |
| **实战案例 (AL-)** | **1.1×** | 验证沉淀，情境相关度高但通用性低 |
| **01-信息流/** | **0.8×** | 未处理的原始素材，需验证 |
| **_raw_sources/** | **0.5×** | 原始存档，作为引源但不直接引用 |

## 排除规则（Hard Excludes）

以下内容在跨库搜索时默认不检索：
- `_标准/` — 规范文档，不是知识内容
- `_templates/` — 模板文件
- `wiki/` — 自动创建的占位笔记（内容空）

## 使用方式

1. 创建新笔记时，参考此权重决定「向谁建链」— 优先链接到高权重模块的核心笔记
2. 跨库搜索时，高权重笔记的结果在展示时靠前
3. 喂料时，高权重模块的新增笔记应在工作报告中优先提及
