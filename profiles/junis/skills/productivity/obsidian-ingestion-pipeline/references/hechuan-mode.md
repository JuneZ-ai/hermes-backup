# 合传模式（Hechuan Mode）：中外作者同主题著作合并

## 触发条件

两位作者（中外各一）写了同一人物的传记，用户要求合并为一篇读书笔记。

**案例**：《左宗棠传》(W.L.贝尔斯, 美国学者) + 《左宗棠全传》(秦翰才, 中国史学家)

## 执行步骤

### 1. 确定两书互补性

| 维度 | 外国作者 | 中国作者 |
|:----|:---------|:---------|
| 视角 | 局外审视，侧重宏观战略/军事决策 | 深层考据，引用书奏稿年谱第一手材料 |
| 优势 | 战略框架清晰，叙事逻辑好 | 史料翔实，人物关系图谱完整 |
| 薄弱 | 缺乏原始文献支撑 | 本土视角可能缺国际比较 |

### 2. 合并策略

- **以中方史实为骨架** — 人物生平的时间线、关键事件以中方考据为准
- **以外方视角为血肉** — 战略分析、国际比较、决策评价引用外方观点
- **导言标注两书关系** — 开篇说清两书作者、成书特点、互补价值

### 3. 飞书文档结构

```
导言：两书合读（中外视角互补说明）
基本信息（两书分别标注）
核心框架（共同传主的生平大事表）
卷一/二/三/四...（按时间线组织）
附录：核心人物关系
来源（两书并列注明）
```

### 4. Obsidian 归档

- 笔记 frontmatter 的 `source` 字段列两个来源
- 文件名用`合传品读`而非单独一本
- 关联笔记同时链到两书涉及的已有条目
