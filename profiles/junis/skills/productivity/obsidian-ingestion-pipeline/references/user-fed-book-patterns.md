# 用户投喂教材/整书——框架笔记模式参考

## 场景

用户通过飞书直接发送 Markdown 文件（教材/书籍/经典著作），要求收录原文+创建框架笔记。本文件记录已验证的笔记结构模式。

## 模式A：西方哲学/伦理学经典（支柱型笔记）

**示例**：DS-18-亚当·斯密道德情操论.md

适用条件：
- 著作以概念体系而非章节递进为骨架（如「同情」「公正旁观者」「美德」）
- 适合用"核心支柱"组织而非逐章提炼

### 笔记结构

```
YAML frontmatter
  - title: 描述性的完整标题（含副标题提炼核心）
  - tags: 六韬史鉴 + 领域标签（道德哲学/伦理学/经济学等）+ 作者
  - aliases: 中英文别名
  - created/updated
  - source: 原文链接 [[古籍全文/书名]]
  - version: v1.0
  - links: 原文链接 + 关联DS笔记

一、书籍定位
  1.1 作者与背景（150-300字）
  1.2 全书总纲（章节→主题→核心问题 对照表）
  1.3 一句话定位

二、核心框架：N大支柱
  支柱一：核心概念名
    - 核心命题（作者原话/定义）
    - 理论要点（3-5条，明确mechanism）
    - 重要推论（表格形式）
    - 跨模块映射（→DS-14/DS-17等）
  支柱二：同格式
  支柱N：同格式

三、核心概念速查（表格：概念|定义|现实映射）

四、跨模块关联
  4.1 与作者另一代表作的关系（如斯密两书统一）
  4.2 与知识库已有思想家的对比（表格：3+维度）
  4.3 与马哲/王阳明等已有框架的对话

五、个人阅读笔记
  - 最值得深读的章节
  - 核心洞察

六、来源信息
  - 作者/书名/译者/出版社/版本
  - 原文存放：[[古籍全文/书名]]

七、关联模块（2-3个wikilink）
```

### 关键特征

| 要素 | 说明 |
|------|------|
| 支柱数 | 3-5根，过多则分类不当 |
| 概念速查表 | 必选——让后续快速检索概念定义 |
| 跨模块对比 | 必选——必须与至少2个已有DS笔记对话 |
| 作者两书统一 | 如有姊妹篇，必单独成节（如斯密问题） |

## 模式B：教材/教科书（章节型笔记）

**示例**：DS-14-马克思主义哲学原理.md（v2.5+更新）

适用条件：
- 教科书结构，逐章递进
- 每章有明确的学习目标和知识体系

### 笔记结构

```
一、书籍定位
  1.1 教材在学科中的位置
  1.2 全书章节总览表

二、核心支柱（按主题而非章节聚合）
  支柱一：主题名（跨章节提炼）
  支柱N：同

三、经典案例/应用

四、横向对比（vs知识库中其他思想家）

五、对普通人的启示
```

## 通用规则

| 规则 | 说明 |
|------|------|
| **DS编号** | 扫描 `六韬史鉴/` 中现有 `DS-`前缀文件，取最大序号+1。不跳过断号。 |
| **source字段** | 原文入库后，source 中用 `[[古籍全文/书名]]` 建立双向链接 |
| **links字段** | frontmatter 中加 `links:` 列出原文+关联笔记的完整wikilink |
| **跨模块对比** | 引用的已有DS笔记必须在对应笔记的「关联阅读」区注入反链 |
| **不存原文则source注明** | 用户没说要原文，source写「用户已提供原文文件，未要求入库」 |
