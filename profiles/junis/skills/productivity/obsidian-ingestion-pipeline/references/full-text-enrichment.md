# Full-Text Enrichment Pattern (原文血肉化)

## 概念
将一个已存在的深研版笔记（v2.0）用完全集/全书的原文进行"血肉化"升级，让每个论点都有第一手原文支撑。

## 前置条件
- 目标笔记已存在且为 v2.0 深研版（500±行量级，有框架无原文）
- 用户持有该思想家的完整作品集 PDF（非扫描版，= pymupdf 可提取文本）
- 笔记的章节结构稳定（不应在原文注入过程中改变框架）

## 完整工作流

### Phase 1: PDF 全量提取
```bash
import pymupdf
src = pymupdf.open('path/to/pdf')
full_text = ''
for i in range(src.page_count):
    full_text += src[i].get_text() + '\n'
total_chars = len(full_text)
# 验证提取质量：每页应 > 50 字（否则为扫描版）
```

### Phase 2: 原文勘探（Key Passage Discovery）
对现有笔记的关键章节/论点进行反向搜索，寻找全集中的对应原文：

```python
# 对笔记中的每个核心论点，搜索全集中的原文位置
searches = ['心即理', '知行合一', '致良知', '事上磨练', ...]
for s in searches:
    idx = text.find(s)
    if idx >= 0:
        # 提取上下文（前后200-300字）
        snippet = text[max(0,idx-200):min(len(text),idx+400)]
```

**勘探维度：**
- 笔记中的口号/金句 → 搜索原文全文
- 笔记中概括的思想 → 搜索对应章节的原文对话
- 笔记中转述的故事 → 搜索故事原典的完整叙述
- 笔记中提到的概念 → 搜索该概念在全集中的系统论述

### Phase 3: 原文采集（Curated Extraction）
将找到的原文组织成结构化参考文件：

```
| # | 原文段落 | 出处 | 插入位置 |
|---|---------|------|---------|
| 1 | "..." | 传习录上·徐爱录 | 2.1 心即理 |
| 2 | "..." | 传习录中·答顾东桥书 | 2.2 知行合一 |
```

**关键原则：**
- 只提取与笔记已有结构对应的原文（不引入新框架）
- 每段原文控制在 200-800 字之间
- 优先找经典问答/对话（传习录式的"问-答"结构最有感染力）
- 确保原文有明确的出处标注（篇目·节名·对话人）

### Phase 4: 注入（Injection）
使用子代理（delegate_task）或手写方式将原文注入笔记：

**注入规则：**
1. **前置引用** — 在现有一点论的"白话解读"之后，追加"原文原文证实"
2. **不替换** — 白话解读保留，原文作为证据补充
3. **格式** — 使用引用块（>）标注原文，末尾标注出处
4. **链接** — 注入原文后，更新 YAML 版本号（v2.0 → v3.0）
5. **source 字段** — 更新为完整作品集名称

### Phase 5: 验证
```bash
# 检查原文引用数量
grep -c '^>' 笔记路径
# 检查重要段落是否加入
grep '经典原文关键词' 笔记路径
# 检查版本号
head -20 笔记路径 | grep 'version:'
```

## 实际案例：王阳明 v2.0 → v3.0

| 指标 | v2.0 | v3.0 | 增量 |
|:----|:----:|:----:|:----:|
| 行数 | 305 | 452 | +147 |
| KB | 15 | 28 | +13 |
| 原文引用 | ~5条（凭记忆） | 21条（全集原文） | +16条 |
| 来源 | 王阳明全集(377p/25.7万字) |

## 不适用条件

**不适用：**
- 纯信息报告/数据分析笔记（不需要原文支撑）
- 扫描版 PDF（无文本可提取）
- v1.0 笔记（应先完成深研版 v2.0 框架）

## 双书交叉注入（Dual-book Enrichment）

当用户有**同一思想家的两本著作**，需要同时注入现有笔记时，流程与单书注入有显著差异。

### 元数据陷阱
- 信息流笔记标记为 `status: processed` **未必意味着全量提取已完成**
- 如果素材笔记只有 11-12 行（YAML frontmatter + 一句话摘要），说明原始处理时**跳过了全量提取**
- 原始PDF可能仍在 `/home/hermes/.hermes/cache/documents/` 缓存中
- **验证方法**：`ls -la` 检查缓存中是否有同名PDF

### 勘探差异
勘探不再按单本书的关键词搜索，而是按**主题**在**两本书中分别搜索**：

```python
# 主题驱动的双书勘探
themes = [
    ('圆圈正义', '圆比喻|画圆|三种|完美之圆'),
    ('人的有限性', '有限|幽暗|不确定|经不起考验'),
    ('次优选择', '次优|最不坏|最优|地狱之路'),
    # ...
]

for theme_name, keywords in themes:
    # 圆圈正义中搜索
    idx1 = text1.find(keywords)
    # 法治的细节中搜索
    idx2 = text2.find(keywords)
```

### 引用标注规范
双书注入时，每条原文引用必须标注**具体书名**（含文章名），格式：
```
"原文正文" ——《圆圈正义·圆圈正义与安提戈涅》
"原文正文" ——《法治的细节·法治如何在不确定中寻找确定性》
```

### 金句集增强
单书注入：5条 → 15条（从原有框架扩展）
双书注入：12条 → 26条（从两本中分别取精华，避免偏重某一本）

### 对比表
延伸阅读区新增**双书对比表**：从出版年份、风格、侧重点、代表性文章、哲学深度、对知识库的贡献 6个维度对比两本书。这是双书注入的独特产出。
