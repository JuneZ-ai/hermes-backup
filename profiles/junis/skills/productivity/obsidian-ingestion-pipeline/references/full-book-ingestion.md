# 大型PDF书籍深度入库参考

> 基于《穷查理宝典》305页/29.5万字的入库实践

## 一、全过程时序图

```
PDF到达 (用户发送附件)
    │
    ├── Phase 1: 元数据检查 (~30s)
    │   ├── fitz.open() → 页数/文本层/图片
    │   ├── 目录页扫描 → 章节结构确认
    │   ├── 逐页文本量扫描 → 扫描版判定
    │   └── 统计: 总分符、文本页/无文页码
    │
    ├── Phase 2: 全量提取 (~10s)
    │   ├── 逐页 get_text() → JSON保存(含页号)
    │   └── txt保存(纯文本)
    │
    ├── Phase 3: 子代理并行分析 (~2min)
    │   ├── Agent A (核心方法论+思维模型)
    │   │   └── 输出: 思维模型清单+决策框架+原文定义
    │   ├── Agent B (特色专题: 25种心理倾向)
    │   │   └── 输出: 逐条提取+跨知识库对比
    │   └── Agent C (生平+金句+哲学)
    │       └── 输出: 生平关键时刻+48条金句+跨思想家对比
    │
    ├── Phase 4: 融合创作 (~1min)
    │   ├── 读三个子代理输出
    │   ├── 按结构框架拼接
    │   ├── 补充跨模块对话(5-7组)
    │   └── 写 frontmatter + 全量笔记
    │
    ├── Phase 5: 总纲三处更新
    │   ├── 模块总纲(00-史鉴总纲.md): 模块地图 + 关联表
    │   ├── 主总纲(00-六韬总纲.md): 树 + 计数 + 场景
    │   └── 反向链接注入(3-5条)
    │
    └── Phase 6: 飞书三表同步
        ├── 搭建日志: 新增1条
        ├── 每日记录: 更新当日内容
        └── 收藏随想录: 仅外部素材
```

## 二、核心代码片段

### 2.1 全量提取与JSON保存

```python
import fitz, json

doc = fitz.open("书.pdf")
all_pages = []
for i in range(doc.page_count):
    text = doc[i].get_text().strip()
    all_pages.append({"page": i+1, "text": text})
doc.close()

# JSON结构: [{"page": 1, "text": "..."}, ...]
json.dump(all_pages, open("raw.json", "w", encoding="utf-8"), ensure_ascii=False, indent=1)
```

### 2.2 章节关键词定位

```python
for kw in ["关键词1", "关键词2", "核心章节名"]:
    for i in range(doc.page_count):
        text = doc[i].get_text()
        if kw in text:
            print(f"'{kw}' 出现在第{i+1}页")
            break
```

## 三、从本次实践提炼的教训

### 3.1 子代理输出文件命名规范

子代理写出的文件名必须明确，父代理才能找到：

```python
# Agent A 的 output 中明确写路径
write_file("/home/hermes/tmp/书_核心提取.md", content)
# 父代理读:
read_file("/home/hermes/tmp/书_核心提取.md")
```

### 3.2 大文件读取限制

子代理输出28KB+需要分次读取：
```python
read_file("output.md", limit=100)  # 先读开头
read_file("output.md", offset=101, limit=100)  # 续读
```

### 3.3 总纲patch的表格格式验证

总纲中的 `[[笔记\\|别名]]` 格式在 patch 时容易产生 `|||` 三竖线错误。每次 patch 后必须 `read_file` 验证表格区域格式正确。

### 3.4 记忆空间管理

大操作后保存记忆时，如果空间不足（接近2200字符上限），优先清除：
- 已不再需要的工具配置备注（如不可用的PaddlePaddle）
- 过期的用户偏好（已通过技能固化了的）
- 不再相关的环境细节（已修复的网络限制）

保留：
- 知识库结构信息（模块路径、文件命名规则）
- 用户核心偏好（响应风格、交付格式）
- 技能引用（已有技能的路径和使用条件）

## 四、跨模块对话：从"平行知识"到"网络化知识体"

整书入库后的核心增值在于**跨模块对话**的构建。以下是从《穷查理宝典》实践提取的模式化跨模块对话清单：

### 4.1 通用跨模块对话映射表

| 书籍类型 | 自然对话对象 | 对话结构 |
|:---------|:------------|:---------|
| 人性论/心理学 | → 韩非子法哲学 | 系统约束 vs 制度约束：共同结论都是"别指望人变好，用系统保护人" |
| 知行方法论 | → 王阳明心学 | 检查清单 vs 良知：一个向外归因、一个向内归因，但都拒绝纯意志力路线 |
| 调查研究/能力圈 | → 毛泽东选集 | 不碰不理解的能力圈 vs 没有调查就没有发言权：排除法思维 |
| 保留安全边际 | → 邓小平时代 | 安全边际 vs 摸着石头过河：都承认认知局限，都保留容错空间 |
| 多元思维模型 | → 马克思主义哲学 | 多学科模型阵列 vs 辩证法：静态分析工具 vs 动态演进视角的互补 |

### 4.2 跨模块注入检查表

创建每篇整书笔记后，检查以下五条跨模块对话是否自然成立：

- [ ] 与王阳明（知行）— 这本书的"如何做"和王阳明的"致良知"是什么关系？
- [ ] 与韩非子（人性）— 这本书对人性的假设是什么？和韩非子的"好利恶害"一致还是对立？
- [ ] 与毛泽东（实践）— 这本书的方法论和毛的"调查研究/矛盾论"有共鸣吗？
- [ ] 与马哲（规律）— 这本书认识世界的方式和辩证法互补还是竞争？
- [ ] 与邓小平（灰度）— 这本书对不确定性的处理方式可以和"摸着石头过河"对话吗？

这条检查表确保每本入库的书都与知识库中至少5条已有神经突触连接，避免孤立节点。
