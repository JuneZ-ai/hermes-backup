# Bitable 监控字段模式（Monitor Field Pattern）

> 在飞书多维表格中新增监控字段，用于跟踪操作/流程中的可观测信号。核心是：把抽象概念转化为可统计的标签。

## 适用场景

- 想监控某个维度的变化趋势（如沟通顺畅度、协作卡壳率）
- 现有数据中已有操作记录，但缺少一个维度来标注"状态"
- 需要定时生成趋势报告（每日/每周）

## 模式步骤

### Step 1：新增字段

在目标 Bitable 表中新增一个 SingleSelect 字段，选项数控制在 3-5 个。

原则：
- 选项名用 emoji 前缀区分视觉风格（该用户偏好：统一空格 + emoji 风格）
- 选项数不要太少（<3 无法区分状态）也不要太多（>5 难以选择）
- 每个选项含义互斥，不要有模糊地带

示例（本会话中的"沟通状态"字段）：
```
选项: ✅ 顺畅 / ⏳ 卡壳 / ○ 无需沟通
```

### Step 2：回填历史数据

通过飞书 API 拉取所有现有记录，用关键词规则智能回填：

```python
# 示例：基于关键词的智能回填
卡壳_keywords = ["驳回", "失败", "卡壳", "不通", "修复"]
for rec in records:
    content = rec["fields"].get("操作内容", "")
    if any(k in content for k in 卡壳_keywords):
        tag = "⏳ 卡壳"
    else:
        tag = "✅ 顺畅"
    # PUT 更新记录
    requests.put(f"{base}/records/{rec_id}", json={"fields": {"沟通状态": tag}})
```

在每日/每周巡检脚本中加入监控维度分析：

### Step 4：设置定期报告

配置 cron job，周五出周报：

```
cronjob(
    action="create",
    schedule="0 18 * * 5",  # 每周五 18:00
    script="ontology_mapper.py",  # 数据收集脚本
    prompt="生成《XX周报》，分析趋势变化..."
)
```

## 本会话案例：燕青说得着检测官

### 做了什么

在「搭建日志」表新增「沟通状态」字段（3选项），回填38条历史记录（基于关键词智能分类），升级本体巡检脚本加入沟通分析，设置每周五周报。

### 第一周基线数据

```
✅ 顺畅: 30 条 (79%)
⏳ 卡壳: 1 条 (3%)  → 集中在知识库基建领域
○ 无需沟通: 7 条 (18%)
健康基线: 正常
```

### 可复用到其他场景

| 场景 | 监控对象 | 字段名 | 选项 | 回填规则 |
|:----|:--------|:------|:----|:--------|
| 沟通质量 | 搭建日志 | 沟通状态 | 顺畅/卡壳/无需沟通 | 关键词匹配 |
| 项目健康度 | 项目看板 | 健康状态 | 🟢正常/🟡风险/🔴阻塞 | 基于截止日期和依赖关系 |
| 知识处理 | 收藏随想录 | 处理状态 | 已收藏/正在读/已入库 | 基于状态字段变化 |
| 协作效率 | 日程/任务 | 协作复杂度 | 单人/双人/跨团队 | 基于参与人数 |

## 注意事项

- 回填数据时不要用 PUT 直接替换字段选项（会导致已有记录的选项 ID 失效→值清空）
- 必须先迁移记录（PUT record 传选项 NAME），再更新字段定义
- 每周报告的 prompt 要具体（不是"分析数据"，而是明确输出结构）
