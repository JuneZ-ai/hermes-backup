# Vault 架构重构指南

## 核心理念

大规模 vault 重构遵循以下原则（来自花叔橙皮书 + 六韬实操）：

1. **不搬文件只改总纲** — 物理文件不要动，只改逻辑归属（总纲树结构+分类标注）
2. **双归属** — 跨模块内容不强制二选一，同时属于两个模块是正常状态
3. **前缀系统消除冲突** — AA-NN- 命名使每个模块独立计数，避免全局冲突

## 架构模型：一棵树，不是一栋楼

2026-05-27 总纲重构后，知识库被定义为「树」而非「建筑」：

```
树冠（行·实战案例）→ 开花结果
  ↑
枝干交汇（融·决断之桥）→ 理觉合一
  ↗            ↖
枝干左（理·智脑）  枝干右（觉·易哲）
  ↖            ↗
  主干（鉴·史鉴）→ 从根系升起的第一道结构
    ↑
🌳 根系（太极双螺旋）→ 底层思想范式
   ├── 向下汲取：中国传统思想 × 辩证唯物·历史唯物
   └── 破土成枝：根系直接长出新枝干（自心洞察/狂热心理学/纳瓦尔幸福）
```

| 建筑模型（旧） | 树模型（新） |
|:---|:---|
| 楼层固定不动 | 树一直在长——根系深扎、枝干伸展、树冠更新 |
| 单向支撑 | 双向循环：根系供养分向上，果实落回土壤 |
| 地基埋在底下不动 | 根系主动生长，还会破土成枝 |
| 楼层间有分界 | 枝干可交叉（决断之桥就是连接处） |
