# Auto-Backlink：日记自动双向链接

> 你写日记，燕青帮你织网。

## 适用场景

你写完当天日记（`YYYY-MM-DD.md`）后，希望AI自动识别其中的人名、书名、概念、专有名词，在vault中搜索已有笔记并添加 `[[wikilinks]]`，没有的自动创建占位实体笔记。

## Vault 信息

- **Vault路径**：`/mnt/c/Users/18502/Documents/Obsidian Vault/`
- **日记命名**：`YYYY-MM-DD.md`，存放在vault根目录
- **知识笔记目录**：`六韬智脑/`、`01-信息流/`、`wiki/`、`太极双螺旋/`、`六韬易哲/`、`六韬史鉴/`、`_raw_sources/`、`决断之桥/`、`团队/`

## 触发方式

### 方式A：手动触发（推荐先跑）
飞书@燕青说：
```
燕青，处理今天的日记
```
或指定日期：
```
燕青，处理 2026-05-26 的日记
```

### 方式B：定时自动（熟悉后启用）
每天固定时间自动扫描当天日记，有变化则通知，无变化静默。

## 工作流

### Step 1：定位日记文件

读取vault根目录下当天的 `YYYY-MM-DD.md` 文件。

### Step 2：读取并分析内容

读取日记全文，识别以下类型的实体：

| 实体类型 | 示例 |
|---------|------|
| 人名 | 段永平、芒格、王阳明 |
| 书名/文章名 | 《反脆弱》、纳瓦尔宝典 |
| 概念/术语 | 本分、平常心、双向链接 |
| 项目名 | OPC、异虎事业部 |
| 地点/机构 | 步步高、OPPO、飞书 |

### Step 3：搜索vault已有笔记

搜索范围包括vault下所有 `.md` 文件，确认每个实体是否有对应笔记。

### Step 4：添加双向链接

对于vault中**已有**笔记的实体：
- 将日记中的纯文本提及替换为 `[[实体名]]`
- 如果笔记有 `aliases`（别名），优先使用别名匹配
- 示例：`今天读了段永平的新文章` → `今天读了[[15-段永平本分商业哲学|段永平]]的新文章`

### Step 5：创建占位笔记（缺失实体）

对于vault中**没有**对应笔记的实体：
- 在 `wiki/` 目录下创建占位笔记 `wiki/实体名.md`
- 格式模板：
```markdown
---
title: 实体名
tags: [wiki, stub]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---

# 实体名

> 自动创建的占位笔记，待补充内容。

## 来源
- 首次出现于：[[YYYY-MM-DD]]
```

### Step 6：输出处理报告

处理完成后回复一份简洁报告，格式如下：

```
📋 日记 backlinks 处理完成

🔗 已链接：2个
  - [[段永平]] → 六韬智脑/15-段永平本分商业哲学
  - [[纳瓦尔]] → 六韬智脑/26-纳瓦尔财富哲学

🆕 新建占位：1个
  - wiki/某某某.md（待补充内容）

⏱ 耗时：XX秒
```

## 🔴 检查点（必须显式标记）

在执行以下操作前**必须暂停并报告**，等待用户确认：

| 检查点 | 触发条件 | 等待确认 |
|--------|---------|---------|
| **CP-1** | 要修改或删除日记的现有 `[[wikilinks]]` 时 | 报告计划变更，用户确认后再执行 |
| **CP-2** | 要创建占位笔记超过 5 个时 | 列出清单，用户确认批量创建 |
| **CP-3** | 遇到同名歧义无法自动判断时 | 列出选项，用户选择目标 |

**格式要求**：使用 `🔴 CHECKPOINT CP-X` 标记，清晰列出选项，等待用户明确回复。

## 注意事项

1. **只修改目标日记文件**，不碰vault中其他笔记
2. **不删除任何内容**，只添加 `[[]]` 结构
3. 遇到同名歧义，优先链接到最相关的目标
4. 日记中已存在的 `[[链接]]` 不做重复处理
5. 占位笔记 tags 固定包含 `[wiki, stub]`
6. **不要改动**日记的 YAML frontmatter 区域
7. 日记为空或不存在时回复「今天没有日记需要处理」
8. 如果日记内容已有`[[]]`，检查是否正确；不正确的链接需要修正
9. **失败处理**：如果vault路径不存在或日记文件无法读取，立即报告错误原因并停止，不要静默失败

## ❌ 反例黑名单（禁止事项）

以下行为会导致vault结构损坏或链接失效，**绝对禁止**：

| # | 禁止行为 | 后果 |
|---|---------|------|
| B-1 | **批量替换前不备份日记** | 误操作无法回滚 |
| B-2 | **删除日记中的现有 `[[]]`** | 已有链接永久丢失 |
| B-3 | **在日记 YAML frontmatter 中添加 `[[]]`** | Obsidian 解析 frontmatter 为元数据，链接不生效 |
| B-4 | **创建占位笔记时不加 `[wiki, stub]` tags** | 后续无法识别哪些是待填充笔记 |
| B-5 | **用 `|别名` 语法但 alias 不存在** | Obsidian 显示红色链接，点击404 |
| B-6 | **修改日记日期或文件名** | `[[YYYY-MM-DD]]` 引用失效 |

**红灯信号**：遇到以下情况立即停止并报告 — vault 路径不存在、日记文件被锁定/只读、占位笔记目录不可写。
