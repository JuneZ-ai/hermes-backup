# Vault 原文覆盖审计方法

> 吸收自 `vault-original-text-audit`（已归档）。此参考文件补充 knowledge-architecture 的 Phase 5（原文覆盖审计）和 Phase 5a（全库入仓审计）的实操细节。

## 原文来源优先级

当发现 vault 缺少原文时，按以下顺序查找：

1. **WB skills texts/** — 先检查 `/mnt/c/Users/18502/.workbuddy/skills/六韬易哲/texts/`
   - 命名格式：`书名-作者.md`
   - 覆盖98+部经典（易经/八字/中医/相学/风水/六壬/奇门等）
2. **外部搜索** — WB没有的再走外部
3. **用户上传** — 用户主动给的PDF

## 优先级分级标准

| 级 | 含义 | 适用 |
|:--:|:----|:-----|
| P0 | 用户当前方向核心 | 易经新方向、历史主干 |
| P1 | 重要补充 | 补充主干方向的经典 |
| P2 | 锦上添花 | 其他模块的补充经典 |

## 入库后处理清单

每部原文入库后必须：
1. ✅ 添加 frontmatter（title/author/tags/category/source）
2. ✅ 按《期刊排版格式规范》处理标点/段落
3. ✅ 更新模块 MOC 索引中的笔记列表
4. ✅ 更新总纲统计（`00-六韬总纲.md` 中模块篇数/状态）

## Pitfalls

- ❌ 不要一上来就外部搜——先检查WB skills texts/目录
- ❌ 不要只入库不更新索引——MOC和总纲必须同步更新
- ❌ 不要假设小文件就没原文——有些短经典（黄金策2K/增删卜易9K）就是那么短
- ✅ file size是最快筛选器：>50KB必含原文，<5KB必是框架
