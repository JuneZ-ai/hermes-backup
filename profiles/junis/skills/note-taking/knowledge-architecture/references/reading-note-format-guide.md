# 用户偏好的读书笔记格式

基于《左宗棠全传·读书笔记》格式，适用于所有同类读书笔记文档。

## 标准结构

```
📖 基本信息（书名/作者/出版/卷册/一句话定位） ← 必须
🔑 核心框架（核心主题表/框架图） ← 必须
卷/章节点（正文精读）
附录（人物关系/目录）
```

> **⚠️ 用户明确规定：基本信息 + 核心框架 是读书笔记的强制组成部分，缺一不可。** 参考《人类简史》笔记格式。2026-05-29 用户明确要求"以后读书笔记都应该有这两个部分"，已记入memory。

## 飞书文档创建顺序

1. 先创建骨架（title + h1各部分 + 基本信息h2 + 核心框架h2 + 章节标题占位）
2. 若文档大，用 skeleton → parallel agent append 分段填充
3. 最后补充附录/金句等收尾

## 基本信息格式

```markdown
<p><b>书名：</b>标题</p>
<p><b>作者：</b>作者名</p>
<p><b>出版：</b>出版社，年份</p>
<p><b>卷册/页数：</b>...</p>
<p><b>一句话定位：</b>用一句话概括全书核心</p>
```

## 核心框架格式

用表格呈现书的核心理念或框架。如《人类简史》用三场革命表（革命/时间节点/核心机制），毛选用五大支柱表（支柱/核心内涵/代表性篇目）。

## 每篇文章四维结构（毛选风格）

```
写作背景 → 核心论点（原文支撑）→ 方法论价值 → 金句摘录
```

- 核心论点用 `<blockquote>` 高亮最经典原文
- 方法论价值用 `<ul>` 要点列表
- 金句摘录用 `<ol>` 编号列表

## 飞书 XML 呈现偏好

- 每个卷/章节以 `<hr/>` 分隔
- 金句用 `<blockquote>` 引用
- 对比/汇总用 `<table>` + 表头 `<th background-color="light-gray">`
- 关键提示用 `<callout emoji="💡⭐📖📜">` 高亮框
- 诗词原文按原词分行，用 `<br/>` 换行
