# 渐进式信息架构实战：WB Skill 蒸馏案例

> 2026-05-31 对 WorkBuddy 中两个巨型 Skill 进行渐进式重构的实证记录。

## 改造对象

| Skill | 改造前 | 改造后 | 压缩比 |
|-------|--------|--------|--------|
| 六韬易哲 V4.0→V5.0 | 139KB / 3274 行 | 6KB / ~150 行 | **23x** |
| 六韬智脑 V9.1→V10.0 | 68KB / 1116 行 | 4.6KB / ~120 行 | **15x** |

## 改造步骤

### 1. 诊断：找出内联重复

两个 skill 的 SKILL.md 都**已经**有 modules/ references/ 目录存放详细知识，但 SKILL.md 自身又把那些内容**内联抄了一遍**。结果：
- SKILL.md 既是导航页也是百科书 → 3274 行
- 修改时需同步两处 → 版本容易不一致
- agent 加载时全量读入 → 浪费 token

### 2. 提取：把内联内容迁到卫星文档

| 提取内容 | 目标文件 | 原位置（行号范围） |
|---------|---------|------------------|
| System Prompt + 双身份 | `modules/人设.md` | 六韬易哲 L327-398, L571-584 |
| 风格指南 + 断语模板 | `modules/风格指南.md` | 六韬易哲 L401-552 |
| 20 个场景案例 | `guides/scenarios.md` | 六韬易哲 L2799-3263 |
| 四大核心模块 | `references/核心模块详解.md` | 六韬智脑 L116-397 |
| 神经突触联动 | `references/神经突触联动.md` | 六韬智脑 L398-490 |
| 六阶段完整工作流 | `guides/完整工作流.md` | 六韬智脑 L542-836 |
| 灰度决策与政策对齐 | `references/灰度决策与政策对齐.md` | 六韬智脑 L837-875 |
| 话术风格指南 | `references/话术风格.md` | 六韬智脑 L946-1081 |
| 输出格式规范 | `references/输出格式.md` | 六韬智脑 L876-945 |

### 3. 重建：SKILL.md 改造成导航页

导航页内容：
- Frontmatter（保持）
- Quick Start（新增，1 条完整路径）
- 核心概念索引（保留）
- 渐进加载表（保留优化）
- 核心算法/工作流（保留，但仅 3-5 段摘要）
- 文件索引（新增，一目了然）
- 表达原则（保留但精简）

### 4. 新增 RULES.md

硬规则提取，每条编号可引用：

```
#1 闭环验证
#2 事实驱动
#3 不编造 node_id
...
```

## 关键经验

1. **压缩比 15-23x 不是目标，是结果。** 如果你发现 SKILL.md 在 300 行以上，大概率能拆。
2. **已有 modules/ 目录却在内联重复 → 直接砍内联。** 这是最高杠杆的改动（1 行删除 = 1 行瘦身）。
3. **导航页必须有文件索引。** 列出「文件名 → 内容 → 何时读」三列，让 agent 和人类都在 10 秒内找到所需。
4. **RULES.md 是「合同条款」拆出来的。** 如果 SKILL.md 里有「必须」「不得」「禁止」出现 5 次以上，考虑抽成独立 RULES.md。

```yaml
# 导航页 SKILL.md 的最小模板

|------|------|------|--------|

## 📂 渐进加载
| 触发词 | 加载文件 |
|--------|---------|

## 📋 核心工作流
[3-5 行流程总览，每阶段 1 行]

## 📄 文件索引
| 文件 | 内容 | 何时读 |
|------|------|--------|

## RULES.md
[链接到 RULES.md]
```
