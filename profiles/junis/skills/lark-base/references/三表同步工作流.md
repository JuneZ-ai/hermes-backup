# 三表同步工作流

> 当向「搭建日志」写入记录时，必须同步检查「每日记录」和「收藏随想录」是否需要更新。

## 三表关系

| 表 | 表ID | 定位 | 写入时机 |
|----|------|------|---------|
| **搭建日志** | `tbllV9WgN64Zwput` | 每次操作一条记录 | 每次操作完成立即写 |
| **每日记录** | `tblTLllADiUdhL6e` | 每日汇总一条 | 同一天有多条操作时，合并到一条每日记录；或新增一条 |
| **收藏随想录** | `tblydJHMALlK3stv` | 有价值的收藏内容 | 遇到可收藏的文章/工具/思想时 |

## 执行规则

### 1. 优先写搭建日志

每条操作独立一条记录，字段：时间/操作内容/领域/状态/沟通状态/备注。用 bot 身份写入。

### 2. 然后检查每日记录

查询当天是否已有每日记录：

```bash
lark-cli base +record-list --base-token <token> --table-id tblTLllADiUdhL6e --limit 50
```

- **已有当天记录** → 用 `+record-batch-update` 追加或更新记录内容
- **无当天记录** → 用 `+record-upsert` 新建一条

每日记录字段：公历时间/农历/节气/干支/黄历宜忌/记录内容/类型。

### 3. 最后检查收藏随想录

当本次操作涉及以下内容时，在收藏随想录记录：
- 公众号文章/网页内容收藏 → 来源类型=微信文章/网页
- 书籍/PDF入库 → 来源类型=书籍/PDF
- 值得记录的想法/感悟 → 标题+我的感悟

收藏随想录字段：标题/来源链接/我的感悟/金句摘录/来源类型/分类/收藏日期/评分/状态。

## 常见场景

| 搭建日志有操作 | 每日记录是否更新 | 收藏随想录是否更新 |
|---------------|----------------|-------------------|
| 知识库入库新笔记 | ✅ 当天汇总 | ✅ 如果笔记来源是外部内容 |
| 工具配置/修复 | ✅ 当天汇总 | ❌ 除非是工具类收藏 |
| 架构决策 | ✅ 当天汇总 | ❌ 除非有思想感悟 |
| 飞书权限开通 | ✅ 当天汇总 | ❌ |
| 读了一篇文章 | ✅ 当天汇总 | ✅ 文章链接+感悟 |

## 示例流程

```
操作: 入库一篇公众号文章到知识库

1. 搭建日志 +record-upsert
   {"操作内容":"入库《XXX》到六韬智脑/24","领域":"知识库基建",...}

2. 每日记录 → 查当天是否有记录
   有 → 追加内容
   无 → 新建："入库《XXX》+其他操作"

3. 收藏随想录 +record-upsert
   {"标题":"XXX","来源链接":"mp.weixin.qq.com/...","分类":"💻 技术/AI",...}
```
