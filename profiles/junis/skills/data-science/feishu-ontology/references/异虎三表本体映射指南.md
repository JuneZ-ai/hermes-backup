## 一、核心概念

让 AI 看懂你的数据世界，需要三层：

```
┌─────────────────────────────────────┐
│  对象 (OBJECT)  ── 实体是什么        │
│  ├─ 操作日志  ── 每一次搭建动作      │
│  ├─ 每日记录  ── 每一天的时间锚点    │
│  └─ 收藏素材  ── 每一份外部输入      │
├─────────────────────────────────────┤
│  关系 (RELATIONSHIP) ── 实体怎么连  │
│  ├─ 日期关联  ── 日志↔记录          │
│  ├─ 收藏关联  ── 素材↔时间          │
│  └─ 引用关联  ── 素材→知识库        │
├─────────────────────────────────────┤
│  动作 (ACTION) ── 实体能做什么      │
│  ├─ 操作-日志同步  ── 写日记自动补  │
│  ├─ 素材自动入库  ── 读完自动喂料   │
│  ├─ 高质量素材预警 ── 高分优先处理   │
│  ├─ 节气思考联动  ── 天时触发思考    │
│  └─ 燕青本体巡检  ── 每日一致性检查  │
└─────────────────────────────────────┘
```

## 二、三表字段映射

### 表1：搭建日志（38条记录）
| 字段 | 映射角色 | 说明 |
|------|---------|------|
| 操作内容 | **动作描述** | 做了什么事 |
| 领域 | **分类标签** | 五大领域：飞书权限/API打通/知识库基建/架构决策/工具配置 |
| 时间 | **时间键** | 操作发生时间 |
| 状态 | **生命周期** | 已完成/待跟进/计划中 |
| 备注 | **元数据** | 补充细节 |

### 表2：每日记录（5条记录）
| 字段 | 映射角色 | 说明 |
|------|---------|------|
| 公历时间 | **对象键** | 核心锚点，每一天一个实体 |
| 农历 | **文化属性** | 时间的人文身份 |
| 节气 | **自然属性** | 自然节律标签 |
| 干支 | **历法属性** | 历法坐标 |
| 黄历宜忌 | **吉凶指示** | 行动指南 |
| 记录内容 | **活动记录** | 当天发生了什么 |
| 类型 | **活动分类** | 读书/工作/会议/思考/生活/见闻/工具 |

### 表3：收藏素材（13条记录）
| 字段 | 映射角色 | 说明 |
|------|---------|------|
| 标题 | **对象标识** | 素材的唯一ID |
| 来源链接 | **关系** | 指向外部世界的连接 |
| 我的感悟 | **认知动作** | 用户对素材的思考处理 |
| 金句摘录 | **内容提取** | 素材的核心精华 |
| 来源类型 | **媒介分类** | 微信/网页/书籍/播客等 |
| 分类 | **领域分类** | 技术/AI/商业/认知/传统文化等 |
| 收藏日期 | **时间属性** | 何时入库 |
| 评分 | **价值评估** | 1-5星 |
| 状态 | **生命周期** | 已收藏/在读/已读完/待重温/已入库 |
